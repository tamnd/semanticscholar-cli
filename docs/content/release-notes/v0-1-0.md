## What's in it

`s2` reads Semantic Scholar through the public Graph API at
`api.semanticscholar.org/graph/v1`. No API key is required.

### Commands

| command | what it does |
|---|---|
| `s2 search <q>` | Search papers by keyword |
| `s2 paper <id>` | Show a single paper's details |
| `s2 citations <id>` | Papers that cite this paper |
| `s2 references <id>` | Papers referenced by this paper |
| `s2 author <q>` | Search authors by name |
| `s2 profile <id>` | Author profile with recent papers |
| `s2 recommend <id>` | Papers recommended as similar |

Paper IDs are flexible: Semantic Scholar paperId, DOI (starting with `10.`),
or arXiv ID (format `NNNN.NNNNN`).

### Output formats

`-o table` (default on TTY), `-o json`, `-o jsonl` (default when piped),
`-o csv`, `-o tsv`, `-o url`. Filter columns with `--fields`.

### Rate limiting

The public API allows roughly 100 requests per 5 minutes. `s2` spaces
requests 200ms apart by default and retries on 429/5xx errors.
